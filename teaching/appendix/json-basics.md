# 附录A：JSON 快速入门（写给工程师）

## JSON 是什么？

JSON（JavaScript Object Notation）是一种**纯文本的数据格式**，就像 TXT 文件一样。它用花括号 `{}`、方括号 `[]`、冒号 `:` 和逗号 `,` 来组织数据。

### 类比：JSON 就像工程表格

| 工程概念 | JSON 表示 |
|---------|----------|
| 一个节点的坐标 X=1, Y=2, Z=3 | `{"X": 1, "Y": 2, "Z": 3}` |
| 一个列表 [67, 130, 171] | `{"NODE_LIST": [67, 130, 171]}` |
| 表格中的一行 | `{"NAME": "C50", "STRENGTH": 50}` |

## JSON 的基本规则

### 规则 1：花括号 `{}` 包住一个「对象」

```json
{
  "X": 1,
  "Y": 2
}
```

花括号里面是 **键值对**：`"键": 值`。键是标签，值是数据。

### 规则 2：方括号 `[]` 包住一个"数组"（列表）

```json
[67, 130, 171]
```

数组就是一串数据，用逗号隔开。可以放数字、字符串、甚至对象。

### 规则 3：对象可以嵌套对象

```json
{
  "MATERIAL": {          ← 第一层：材料对象
    "NAME": "C50",        ← 第二层：材料名称
    "PARAM": {            ← 第二层：参数对象
      "STRENGTH": 50     ← 第三层：强度值
    }
  }
}
```

**理解嵌套的关键**：每一层花括号 `{}` 就是一个新的层级。像剥洋葱一样，一层一层往里看。

### 规则 4：数据类型

| 类型 | 写法 | 示例 |
|------|------|------|
| 字符串（文字） | 用双引号包住 | `"C50"` |
| 数字 | 直接写 | `50`, `3.14`, `-1` |
| 布尔（是/否） | `true` 或 `false` | `true` |
| 空值 | `null` | `null` |
| 对象 | 花括号 | `{"X": 1}` |
| 数组 | 方括号 | `[1, 2, 3]` |

### 规则 5：逗号分隔，最后一项不加逗号

```json
{"A": 1, "B": 2, "C": 3}   ← 正确
{"A": 1, "B": 2, "C": 3,}  ← 错误！C 后面多了逗号
```

## MIDAS API 的 JSON 长什么样？

以"创建节点"为例，发给 MIDAS 的 JSON 长这样：

```json
{
  "Assign": {
    "1": {"X": 0, "Y": 0, "Z": 0},
    "2": {"X": 5, "Y": 0, "Z": 0},
    "3": {"X": 5, "Y": 3, "Z": 0}
  }
}
```

**逐层解释：**

- **最外层** `{}`：整个请求的容器
- **`"Assign"`**：告诉 MIDAS "我要分配数据"
- **`"1", "2", "3"`**：每个节点的编号（你自己定义的序号）
- **`"X", "Y", "Z"`**：每个节点的坐标

这就创建了 3 个节点：节点1在原点，节点2在(5,0,0)，节点3在(5,3,0)。

## 怎么把 JSON 发给 MIDAS？

用 Python 的 `requests` 库，三行代码就够了：

```python
import requests
url = "https://127.0.0.1:1102/db/NODE"      # MIDAS 的地址
data = {"Assign": {"1": {"X": 0, "Y": 0, "Z": 0}}}  # 要发的数据
r = requests.post(url, json=data, verify=False)       # 发送！
```
