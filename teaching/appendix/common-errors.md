# 附录B：常见错误速查表

## 错误类型一览

### 1. JSON 格式错误（最常见！）

**症状**：MIDAS 返回 400 错误或完全不响应
**原因**：JSON 语法有问题

| 错误写法 | 问题 | 正确写法 |
|---------|------|---------|
| `{X: 1}` | 键没有加引号 | `{"X": 1}` |
| `{"X": 1,}` | 最后多了一个逗号 | `{"X": 1}` |
| `{'X': 1}` | 用了单引号（Python字典可以，但JSON不行） | `{"X": 1}` |
| `{\"NAME\": \"C50\"}` | 引号前面多了反斜杠 | `{"NAME": "C50"}` |

### 2. 参数层级放错位置

**症状**：MIDAS 不报错但数据没生效
**原因**：把参数放在了错误的 JSON 层级

```json
// ❌ 错误：X, Y, Z 应该包在 NODE 里面
{"X": 0, "Y": 0, "Z": 0}

// ✅ 正确
{"Assign": {"1": {"X": 0, "Y": 0, "Z": 0}}}
```

**记住**：看接口文档中的 `json_path` 字段，它告诉你每参数应该在哪一层。

### 3. 数据类型不对

**症状**：MIDAS 返回错误或数据被忽略

| 错误 | 问题 | 正确 |
|------|------|------|
| `{"X": "1"}` | X 应该是数字，不是字符串 | `{"X": 1}` |
| `{"NAME": C50}` | C50 没有引号，不是字符串 | `{"NAME": "C50"}` |
| `{"FLAG": "true"}` | true 加引号变成了字符串 | `{"FLAG": true}` |

### 4. 必填参数没填

**症状**：MIDAS 返回错误提示缺少参数
**解决**：检查文档中标记为 `required: true` 的参数，确保都填了。

### 5. 忘记 `verify=False`

**症状**：Python 报 SSL 错误
**原因**：MIDAS 使用自签名证书，不是浏览器信任的证书
**解决**：在 `requests.post(url, json=data, verify=False)` 加上 `verify=False`

### 6. 端口号搞错

| 产品 | 默认端口 |
|------|---------|
| MIDAS Civil | 1102 |
| MIDAS Gen | 1202 |
| MIDAS Civil NX | 1302 |
